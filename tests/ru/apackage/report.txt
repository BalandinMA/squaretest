Проблемы:

1. Некорректное хранение длины стороны квадрата при значении, попадающем в интервалы (0:10), (10:20), (20:30).
    Тест: SquareStandardWorkTest.testSideLength()
1.1 на интервале (0:10):
    ожидаемое значение: sideLength
    фактическое значение: sideLength + 0.9
1.2 на интервале (10:20):
    ожидаемое значение: sideLength
    фактическое значение: sideLength - 0.1
1.3 на интервале (20:30):
    ожидаемое значение: sideLength
    фактическое значение: - sideLength

2. Отсутствие проверки на знак при создании экземпляра объекта, длина стороны квадрата не может быть отрицательной.
    Тест: SquareSpecialParametersTest.testNegativeSideLength()

3. Отсутствие проверки на NaN при создании экземпляра объекта.
    Тест: SquareSpecialParametersTest.testNanSideLength()

4. Отсутствие проверки на максимально допустимое значение длины стороны при создании экземпляра объекта.
   В случае создания экземпляра объекта со стороной, большей чем корень из максимально допустимого double,
   пользователь получить значение площади Infinity, что некорректно.
    Тест: SquareSpecialParametersTest.testMaxValueSideLength()

5. Отсутствие проверки на минимально допустимое значение длины стороны при создании экземпляра объекта.
   В случае создания экземпляра объекта со стороной, меньшей чем корень из минимально допустимого double,
   пользователь получить значение площади 0, что некорректно.
    Тест: SquareSpecialParametersTest.testMinValueSideLength()

6. Для получения площади квадрата необходимо использовать метод Math.pow(sideLength, 2), он обеспечит более высокую
   точность вычислений.
    Тест: SquareSpecialParametersTest.testHighPrecisionSquare()

7. При передаче методу equalsSquare в качестве параметра null корректно ожидать false, либо указать о
   возможном NPE в декларации метода.
    Тест: SquareSpecialParametersTest.testNullEqualsSquare()

8. Для сравнения двух значений double в методе equalsSquare необходимо сравнивать разность двух чисел с выбранной
   погрешностью.
